§ 01 · Landscape 2026

Bối cảnh phân mảnh — 3 nhóm framework

Trước khi quyết định, CEO cần thấy bản đồ thị trường rõ ràng. Mọi agent framework đang competing rơi vào một trong ba nhóm dưới đây — biết nhóm nào để hiểu game nào họ đang chơi.

Nhóm 01 · LLM Provider SDK

Provider-native primitives

SDK do chính chủ LLM phát hành — tối ưu cho model nhà nhưng có thể support model khác qua compatibility shim.

  • OpenAI Agents SDK — 19K stars, 10.3M downloads/tháng, Python-first
  • Google ADK — tích hợp sâu GCP / Vertex AI
  • AWS Bedrock Agents — chỉ chạy trên AWS
  • Microsoft Agent Framework — .NET / Azure ecosystem
  • Claude Agent SDK ★ — candidate trong nhóm này, lý do chọn ở §03
Nhóm 02 · Multi-Provider Framework

Provider-agnostic abstractions

Framework wrap nhiều LLM provider sau một abstraction chung. Đa năng nhưng thường thêm overhead và có learning curve dốc.

  • LangChain / LangGraph — 24.8K stars, 34.5M dl/tháng. Klarna · Uber · LinkedIn · JPMorgan
  • CrewAI — 46K stars, role-based multi-agent
  • Mastra — TypeScript-only, Replit + SoftBank đang dùng
  • Vercel AI SDK v6 — best DX cho React/Next.js, 81 providers
  • PydanticAI · AutoGen · Strands — niche cụ thể
Nhóm 03 · Self-hosted Runtime

Open-source self-hosted

Framework cho phép chạy agent trên hạ tầng của bạn, multi-channel messaging, không phụ thuộc cloud vendor cụ thể.

  • OpenClaw ★ — candidate trong nhóm này, lý do chọn ở §04
  • 180,000+ GitHub stars (lớn nhất trong open-source agent space)
  • ClawHub: 5,700+ community skills
  • Multi-channel: Telegram, WhatsApp, Discord, Slack, Signal, iMessage
  • Multi-model: Claude, Kimi, Llama, Gemini — không lock-in
Insight quan trọng cho CEO

Phần lớn so sánh framework anh đọc trên blog đều trộn ba nhóm này lại như thể chúng đang giải cùng một bài toán. Chúng không. Provider SDK giải production agent embedded trong product. Multi-provider framework giải orchestration phức tạp. Self-hosted runtime giải personal automation và data sovereignty. Quyết định đầu tư đúng = chọn một cái cho mỗi tầng cần thiết, không phải chọn một cái duy nhất cho tất cả.

§ 03 · Production Layer

Tại sao chọn Claude Agent SDK

Cho tầng agent embed trong sản phẩm khách hàng dùng. Năm bằng chứng cụ thể, không lý thuyết.

01
Cùng harness chạy Claude Code đang được 1M+ repos dùng daily

Agent SDK không phải framework "viết mới rồi quảng cáo". Nó là cùng tool loop chạy Claude Code — CLI tool 110,000 GitHub stars, 20.8M commits qua 1M+ repository. Khi anh build agent trên SDK này, anh đang đứng trên hạ tầng đã production-tested ở quy mô khổng lồ.

Anthropic đổi tên Claude Code SDK → Claude Agent SDK cuối 2025 chính vì developer đang dùng nó cho mọi domain — research, video, finance — không chỉ coding.
02
MCP first-class — Anthropic chính là người định nghĩa spec

Model Context Protocol là spec mở Anthropic publish, sau đó Vercel AI SDK, Mastra, CrewAI đều add support trong 6 tháng qua. Khi MCP server anh viết hôm nay sẽ work trên Claude Agent SDK và các framework khác, anh có portability tự nhiên — không lock-in toolchain dù chọn Anthropic làm primary.

10,000+ MCP server đã sẵn có trong registry tính đến đầu 2026. GitHub, Slack, Gmail, Shopify, Playwright — cài là chạy.
03
Time-to-Market nhanh nhất trong nhóm provider SDK

Hello-world chạy được trong < 1 giờ. Capstone agent có MCP + subagent + hook + skill ship được trong < 24 giờ tập trung. So với LangGraph (steepest learning curve trong tất cả framework theo cộng đồng), team anh tiết kiệm 2–4 tuần ramp-up — đáng giá lương 1 senior engineer/tháng.

Anthropic Academy có 16 khóa miễn phí kèm certificate. Junior dev có thể xong Claude 101 + Claude Code in Action trong 8 giờ.
04
Cost predictable với Sonnet là default

Anthropic Docs công bố ~$6/dev/ngày trung bình. Với kỷ luật quản context (compaction ở 60%, Sonnet default cho 90% task, Opus reserved cho orchestrator), team anh đẩy được xuống $3–5/dev/ngày. Với LangChain + LangSmith hoặc CrewAI Enterprise, anh chịu thêm $39–99+/tháng platform fee — chi phí cố định không scale theo value tạo ra.

5 nguyên tắc giữ chi phí dưới $5/ngày: default Sonnet · subagent nhỏ hơn parent · compact ở 60% · gitignore file lớn · hard limit trong settings.json.
05
Vendor risk được đánh giá đúng, không tô hồng

Đây không phải SDK perfect. Anthropic-only — model lock-in là thật. Nhưng risk này nhỏ hơn risk team kỹ thuật của anh không ship được. Quality của model Sonnet 4.6 / Opus 4.7 đang dẫn đầu industry trong agentic task. Khi Anthropic có incident (đã xảy ra Jan 2026 OAuth shutdown), nó chỉ ảnh hưởng third-party tools, SDK chính chủ không bị động.

Hedge cụ thể: Claude Agent SDK cho production-facing layer + OpenClaw cho internal automation. Nếu Anthropic có outage, internal automation vẫn chạy. §05 giải thích chi tiết.
§ 04 · Automation + Hedge Layer

Tại sao chọn OpenClaw

Cho tầng tự động hóa nội bộ + hedge vendor risk. OpenClaw không cạnh tranh với Claude Agent SDK — nó làm điều SDK không làm.

01
Self-hosted data sovereignty

Internal automation thường chạm dữ liệu nhạy cảm — email, customer data, tài chính, HR. Self-host trên VPS của anh có nghĩa data không rời khỏi infrastructure của bạn. Với SaaS-only framework, mọi tool call đều đi qua máy chủ vendor — auditable nhưng không owned.

OpenClaw MIT license, Docker container per session, SQLite metadata local. Deploy anywhere Docker chạy được — VPS Vietnam, AWS, Hetzner, on-prem.
02
Multi-channel native — nơi nhân sự đang làm việc

Vietnamese team không sống trong terminal — họ sống trong Zalo, Telegram, Slack, Discord, đôi khi WhatsApp với khách quốc tế. OpenClaw ship multi-channel routing built-in: một agent core, nhiều channel output. Claude Agent SDK không có capability này natively — anh phải xây.

15+ channel adapter sẵn: Telegram, WhatsApp, Discord, Slack, Signal, iMessage, email, webhook, web UI. Một agent reply ở channel nào tùy task type.
03
Multi-model không lock-in — hedge thật sự

OpenClaw runtime cho phép swap Claude → Kimi → Llama → Gemini bằng config change, không phải rewrite code. Khi Anthropic có outage / giá tăng / model regression / OAuth incident (như Jan 2026), internal automation của anh không dừng. Đây là vendor hedge thực sự, không phải lý thuyết.

Provider abstraction trong OpenClaw runtime — flag CLAW_PROVIDER=anthropic đổi sang =openrouter ship sang Kimi/Qwen ngay.
04
Cost predictable — không platform fee

MIT license = $0 framework cost. Anh chỉ trả token API ($20–200/tháng cho personal, scale theo volume). So với Mastra Cloud $99+/tháng, LangSmith $39+/tháng, CrewAI Enterprise tier — anh tiết kiệm $1,000–3,000/năm chỉ riêng tier subscription nếu chạy 5–10 agent internal.

ClawHub 5,700+ community skill miễn phí. CRM integration, scraping pattern, scheduling pattern — đa phần không phải build từ đầu.
05
Open-source không cô đơn — 180K stars community

Risk lớn nhất với open-source là "nếu maintainer bỏ chạy?" OpenClaw đã vượt qua threshold này: 180,000+ GitHub stars là largest open-source agent framework hiện tại; ClawHub có 5,700+ contributor publish skills. Khi anh hit edge case, có khả năng cao đã có người gặp và fix.

OpenClaw Managed Agents (2026) là cloud service do contributor cộng đồng xây dựng — bằng chứng ecosystem đang grow chứ không stall.
§ 05 · The Two-Stack Model

Cách hai SDK complement, không cạnh tranh

Đây là điểm mấu chốt của recommendation. Đa số CEO sai vì cố chọn một framework làm tất cả. Sự thật: hai layer khác nhau cần hai tool khác nhau.

Lớp chia sẻ: MCP server. Tool nào anh viết một lần (Shopify connector, internal CRM connector, Klaviyo writer) sẽ chạy được trên cả hai stack. Đây là lý do quyết định two-stack không nhân đôi chi phí dev — tool layer dùng chung.
Tại sao không chỉ chọn một?

Chọn chỉ Agent SDK: internal automation phải build từ đầu (multi-channel routing, scheduler, persistent state). Mất 2–3 tháng dev và vendor lock-in cao. Chọn chỉ OpenClaw: production agent thiếu Anthropic-quality MCP integration và bị customer hỏi "tại sao agent đôi lúc trả lời sai" — không có warranty từ vendor chính chủ. Chọn cả hai: mỗi tool làm việc nó best-fit, MCP server dùng chung — đây là Pareto-optimal cho operator Việt 2026.

§ 07 · Migration Path

Lộ trình 90 ngày roll-out two-stack

Không cần big-bang migration. Ba giai đoạn 30 ngày, mỗi giai đoạn có deliverable cụ thể CEO check được.

Ngày 0–30
Pilot Claude Agent SDK · 1 use case sản phẩm

Chọn 1 feature hiện tại đang manual hoặc đang dùng prompt-only LLM call. Refactor sang Agent SDK với 1 custom tool + 1 MCP server. Senior dev (1 người) ship trong 30 ngày. Mục tiêu: có production agent thật chạy với traffic thật, đo được latency và cost/request.

Deliverable · Production agent v1
Ngày 31–60
Setup OpenClaw · 3 internal automation

VPS Hetzner / VPS Việt Nam, deploy OpenClaw qua Docker. Connect 3 channel: Telegram cho team Operations, Slack cho Engineering, email webhook cho CEO. Build 3 use case: daily competitor SERP brief, weekly TikTok Shop top viral product, monthly cost dashboard. Tận dụng MCP server đã viết ở giai đoạn 1.

Deliverable · 3 internal agent live
Ngày 61–90
Scale + Hedge test

Mở rộng Agent SDK production layer thêm 2 feature mới. Mở rộng OpenClaw lên 5–7 internal automation. Quan trọng nhất: simulate Anthropic outage — switch OpenClaw provider sang Kimi/Qwen, verify internal automation vẫn chạy. Đây là vendor risk drill thực sự, không lý thuyết. Cuối quý: Cost report, ROI report, hiring plan year 2.

Deliverable · Vendor risk drill PASS
Budget thực tế cho 90 ngày này

1 senior dev (full-time, $2,000–4,000/tháng VN) × 3 tháng = $6,000–12,000. Token cost trong giai đoạn pilot ~$300–600/tháng × 3 = $900–1,800. VPS cho OpenClaw $20–50/tháng × 3 = $60–150. Tổng đầu tư 90 ngày: $7,000–14,000 để có hai-stack agent production-ready. So sánh: chi phí thuê 1 nhân viên junior thay agent này làm tay = $2,000–3,000/tháng × 12 = $24,000–36,000/năm — và không scale được.

Decision Made · Move Now

Quyết định two-stack không cần đợi.

Mỗi tháng anh trì hoãn = 1 tháng đối thủ tiến lên. Mỗi tháng anh chọn nhầm framework = 2–3 tháng tech debt phải refactor sau. Quyết định đúng hôm nay: Claude Agent SDK + OpenClaw. Triển khai bắt đầu Monday.
